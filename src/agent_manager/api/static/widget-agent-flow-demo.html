Widget → real agent → sub-agent flow

This page loads the real widget against the real conversation API. Nothing on this page is mocked. A question you type travels:

widget → /conversations → agent engine → concierge_router → a sub-agent → answer.

1. Run the backend with the demo config

agent-manager --config examples/widget_sub_agent_demo.yml --env .env --port 8100

Then open http://127.0.0.1:8100/widget-agent-flow-demo.html.

2. Open the widget (bottom-right) and try

  • “show me the available document tags” → routes to tags_agent
  • “what are your support hours?” → routes to hours_agent

3. What proves the sub-agent ran

The backend returns the routing path in the response. The panel below is filled live from the agent-chat:answer event the widget emits after each reply. A hop like concierge_router/tags_agent means the root orchestrator routed to that sub-agent. The same path is logged to the browser console and printed in the server logs (run ended … visited=…).
